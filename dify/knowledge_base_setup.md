# 知识库配置指南

本文档说明如何在 Dify 中配置知识库，使 RAG 检索效果最优。

## 预处理效果

| 报告 | 原始页数 | 提取页数 | 噪音清理率 |
|------|---------|---------|-----------|
| 2025 年度报告 | 282 | 40 | 86% |
| 2025 中期报告 | 122 | 37 | 70% |

保留的 7 个核心章节：
1. 财务概要 / 财务表现摘要
2. 主席报告
3. 管理层讨论与分析（MD&A）— 最核心
4. 综合收益表
5. 综合财务状况表
6. 综合权益变动表
7. 综合现金流量表

## 上传策略

| 策略 | 上传文件 | 优点 | 适用场景 |
|------|---------|------|---------|
| 合集 PDF | `*_核心章节合集.pdf` | 一次上传，保留排版 | 快速验证 Demo |
| 分章节 Markdown | 逐个上传各章节 `.md` | 分段更精准 | 生产环境 / 精细调优 |
| 拆分 PDF (<10MB) | `*_Part1.pdf` 等 | 满足文件大小限制 | Dify 单文件上传限制 |

建议先用合集 PDF 快速跑通 Demo，效果满意后再切换到 Markdown 分章节上传做精细调优。

## 分段设置

| 参数 | 推荐值 | 原因 |
|------|-------|------|
| 分段规则 | **自定义** | 财报结构复杂，自动分段容易截断表格 |
| 分段最大长度 | **1000 Tokens** | 财务分析需要较长上下文理解因果关系 |
| 分段重叠长度 | **100-150 Tokens** | 避免跨页表格被切断后丢失上下文 |
| 文本预处理 | 清理多余空白、URL；关闭 Markdown 转换 | 提取的文本格式已清理，无需额外转换 |

## 检索设置（最关键）

| 参数 | 推荐值 | 原因 |
|------|-------|------|
| 检索模式 | **混合检索 (Hybrid Search)** | 同时需要语义理解和关键词精确匹配 |
| Top K | **5-8** | 财报数据分散在多个段落 |
| Keyword Weight | **0.7** | 财报中精确数字和专有名词多 |
| Semantic Weight | **0.3** | 语义理解辅助 |
| Rerank 模型 | **bge-reranker-v2-m3** 或 Cohere Rerank | 重排后精准度显著提升 |
| Rerank Top N | **3-5** | 只保留最相关的片段 |

## Embedding 模型

| 模型 | 语言支持 | 推荐度 |
|------|---------|-------|
| `bge-large-zh-v1.5` | 中文优化 | 首选 — 腾讯财报为繁体中文，bge 中文理解力最强 |
| `bge-m3` | 多语言 | 备选 — 支持繁体中文+英文混合 |
| `text-embedding-3-large` | 多语言 (OpenAI) | 可选 — 效果好但需 API key |

## System Prompt 建议

在 Dify 的 LLM 节点中加入以下约束：

1. 必须基于知识库内容回答，若知识库中无相关数据，请明确说明
2. 引用数据时，请标明数据源自哪份文档
3. 使用 Markdown 表格形式展示核心财务指标
4. 保持客观、专业、简练的投研语调
5. 当用户问超出知识库时间范围的数据时，说明现有文档覆盖时间段
